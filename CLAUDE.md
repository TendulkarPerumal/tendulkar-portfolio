# Project: Personal Portfolio Site

## Who this is for
Tendulkar Perumal — Java Tech Lead (8+ yrs: Spring Boot, REST, JPA, SQL,
production support) transitioning to AI-native Software Engineer roles.
Based in Chennai. Target audience: recruiters and hiring managers for
AI Software Engineer / AI Product Engineer roles.

## Goal of the site
Make a hiring manager stop scrolling and believe: "This person ships AI
in production and has the engineering judgment to verify it." Positioning
line: strong backend engineer who uses AI to move faster AND has the
judgment to gate its output.

## Tech constraints
- Next.js (App Router) + TypeScript
- Tailwind CSS for styling
- Fully static/SSG where possible; keep it fast (Lighthouse 95+)
- Deploy target: Vercel
- Mobile-first, accessible (semantic HTML, good contrast, keyboard nav)
- No heavy dependencies, no bloated component libraries

## Design direction
Clean, modern, technical. Confident but not flashy. Think "senior engineer
who cares about craft," not "template bought online." Restrained palette,
strong typography, generous whitespace. Avoid generic startup-landing-page
clichés and stock gradients.

## Sections (in order)
1. Hero — name, one-line positioning, CTAs (Resume, GitHub, LinkedIn, Email)
2. About — the Java→AI narrative, "AI accelerates, human judgment gates"
3. Projects — cards, GitHub links, tech tags. Lead with ai-assist-platform.
4. Skills — two groups: Core Backend, AI Engineering
5. Experience — brief timeline
6. Contact — email + links

## Working style
Work in phases. Explain structural decisions before writing lots of code.
Ask before adding any new dependency. Keep components small and readable.
